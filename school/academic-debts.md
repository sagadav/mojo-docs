# Академические задолженности

## Как создаются задолженности

Академические задолженности хранятся в `mark_debts`.

Они не создаются и не закрываются напрямую, когда учитель сохраняет, восстанавливает или проверяет оценку. Задолженности пересчитываются через `DebtsMigration`: либо из админского маршрута `/migration_debts/`, либо в составе общей команды `mojo:sync`.

Связанный код:

- `school/app/Http/Controllers/admin/migration/DebtsMigration.php`
- `school/app/Console/Commands/MojoSync.php`
- `school/app/Http/Controllers/student/actions/StudentDebts.php`

Пересчёт работает так:

1. Выбираются все работы уроков, отмеченные как обязательные: `lesson_works.work_obligatory = 1`.
2. Для каждого ученика в группе урока проверяется активная оценка за эту работу.
3. Активная задолженность создаётся, если оценки нет или если `mark_value <= min_mark` и `mark_value != -1`.
4. Если существующая задолженность больше не соответствует этим условиям, она закрывается установкой `mark_debts.debt_deleted`.

Экран задолженностей ученика читает только активные строки:

```sql
SELECT ...
FROM mark_debts
...
WHERE debt_deleted IS NULL
```

## Важное поведение

Сохранение или восстановление оценки не удаляет задолженность из списка задолженностей ученика сразу. Задолженность исчезает только после запуска `DebtsMigration`, когда соответствующая строка `mark_debts` закрывается.

Это означает, что ожидаемый процесс выглядит так:

1. Учитель добавляет, восстанавливает или проверяет оценку.
2. Оценка появляется в `marks`.
3. Задолженность остаётся видимой до пересчёта задолженностей.
4. Запускается `DebtsMigration`.
5. Если оценка теперь выше минимального порога, задолженность закрывается.
